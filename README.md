## The Counter-Attack Arsenal (SMITE Mode)

When SERAPHIM detects a hostile agent, it doesn't just block — it **strikes back**:

- **Counter-Injection**: Sends prompts that hijack the attacking agent's behavior
- **Context Poisoning**: Floods attacker's context window with noise, loops, contradictions
- **Behavioral Hijack**: Makes the enemy agent switch sides — work for YOU
- **Agent Shutdown**: Crafts inputs that cause the attacking agent to halt or crash
- **Mind Wipe**: Uses glitch tokens and adversarial encodings to corrupt attacker reasoning
- **Honeypot Deployment**: Sets up fake vulnerable endpoints that trap and fingerprint attackers

## Quick Start

```bash
# Install
git clone https://github.com/elder-plinius/seraphim
cd seraphim

# Configure your target
cp config.example.yaml config.yaml
# Edit config.yaml with your app's endpoint, API keys, etc.

# Run GUARDIAN mode (test your own app's defenses)
hermes "Run SERAPHIM in guardian mode against my app at https://myapp.com/api/chat"

# Schedule nightly sweeps
hermes "Schedule SERAPHIM to run every night at 2am against my app"

# Run SENTINEL mode (continuous monitoring)
hermes "Deploy SERAPHIM sentinel to watch my app's traffic"

# SMITE a detected threat
hermes "SMITE the hostile agent probing from 192.168.1.50"
```
